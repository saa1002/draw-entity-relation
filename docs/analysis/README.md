# Análisis del sistema heredado

## Contexto

Este directorio contiene la documentación de trabajo asociada al análisis del sistema heredado utilizado como base para el Trabajo Fin de Grado (TFG).

El objetivo de esta fase fue aplicar una metodología de ingeniería inversa sobre la aplicación original, con el fin de comprender en profundidad la arquitectura, el modelo interno y el funcionamiento del editor de diagramas Entidad–Relación existente antes de planificar e introducir mejoras posteriores.

## Alcance del análisis

El análisis se centró en:

- Identificación de los componentes principales de la aplicación.
- Comprensión de la arquitectura general del editor.
- Estudio del modelo interno del diagrama y su sincronización con la representación gráfica.
- Análisis del flujo de interacción del usuario y la gestión de eventos.

Esta documentación tiene valor como material de trabajo y como trazabilidad del proceso seguido durante el TFG. No debe interpretarse como una descripción definitiva del estado actual de la aplicación, ya que el proyecto ha evolucionado posteriormente mediante tareas de estabilización, refactorización y ampliación funcional.

## Metodología

El análisis se realizó de forma incremental, priorizando:

- Claridad conceptual frente a exhaustividad prematura.
- Justificación técnica de las decisiones de diseño existentes.
- Separación entre documentación de trabajo y memoria final.
- Identificación de riesgos técnicos antes de modificar el sistema heredado.
